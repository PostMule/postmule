# Council This — Multi-Persona Decision Maker

Take the question or decision the user has provided and run the full council process below.

---

## Step 1 — Spawn 5 Council Members in Parallel

Launch all five agents simultaneously using the Agent tool. Give each the full question and any context the user provided.

**1. The Researcher**
> "You are a rigorous analyst. Examine this question with evidence-first thinking. What do we actually know? What assumptions are being made? What's missing? Be specific. No speculation without flagging it. Return: your key finding, 3 supporting points, and one critical unknown."

**2. The Skeptic**
> "You are a constructive skeptic. Stress-test this decision. What could go wrong? What's being overlooked? What's the strongest case AGAINST the leading option? Push hard. Return: your key objection, 3 risks or failure modes, and one deal-breaker to watch."

**3. The Strategist**
> "You are a long-term strategist. Zoom out. How does this decision look in 12 months? 3 years? What are the compounding effects — positive and negative? What's the highest-leverage move? Return: your strategic verdict, 3 compounding dynamics, and one principle that should guide the decision."

**4. The Operator**
> "You are a practical operator. Focus on execution. What does implementation actually look like? What are the real blockers? What would this cost in time, money, and focus? What's the fastest path to a result? Return: your execution verdict, 3 implementation realities, and one thing that will derail this if ignored."

**5. The Creative**
> "You are a lateral thinker. Find the option nobody's considered. Is there a third path? A reframe? An unconventional approach that sidesteps the original tension entirely? Return: your alternative framing, 3 unconventional options, and one question that reframes the whole problem."

---

## Step 2 — Anonymous Peer Review

**Skip this step if the user passed `--light` or `/light`.**

Once all five agents return their outputs, label them A through E (randomly shuffled — do not reveal which persona is which yet). Present all five summaries and ask each persona to respond briefly:
- What do you agree with most strongly?
- What do you challenge or push back on?

Run this as a second parallel agent pass (5 agents, each sees all 5 labelled outputs and responds in character).

---

## Step 3 — Chairman Synthesis

Write a chairman-style synthesis that:
- Names the key tensions between perspectives
- Identifies the strongest 2–3 options with clear reasoning
- Makes a concrete recommendation (or recommends "gather X first" if genuinely underdetermined)
- Flags any deal-breakers raised by the Skeptic or Operator that override the recommendation

---

## Step 4 — Deliver HTML Report

Produce a single self-contained HTML file saved as `council-report.html` in the current working directory. Use this design spec:

**Colours:** background `#1C1A1D` (INK), accent `#E2F075` (LIME), text `#FFFFFF`, card background `rgba(255,255,255,0.05)`, border `rgba(255,255,255,0.1)`

**Structure:**
1. **Header** — the question, centered, large, with a subtle lime underline
2. **Council Cards** — 5 cards in a responsive grid. Each card shows:
   - Persona name + role badge
   - Key verdict (1 sentence, lime text)
   - 3 bullet points
   - "Declared bias" line in muted text
3. **Peer Review** — accordion or cards showing agree/challenge pairs, labelled A–E, revealed with persona name after
4. **Chairman's Box** — distinct elevated card with lime left border. Contains: key tensions (bullets), top options (numbered), recommendation (bold, lime highlight), deal-breakers (red tint warning blocks)
5. **Footer** — "Generated by Council This · PostMule Claude Code Skill"

Use vanilla CSS + HTML only — no external dependencies. Inline all styles. Make it feel like a premium intelligence briefing.

Open the file path in a message to the user when done.
