Pending Reviews

When PostMule sees a new sender name that closely resembles a known entity, it creates a pending match for you to approve or deny.

Approve adds the name as an alias to the matched entity — future mail from that sender will be recognised automatically. Deny dismisses the match; the name remains unrecognised.

Matches not acted on will auto-approve after the configured number of days.

How PostMule Works

PostMule is a self-hosted mail management system. Physical mail scanned by your virtual mailbox service (or biller PDFs sent directly to your email) flows through an AI-powered pipeline that classifies, extracts, and organizes everything automatically.

Components

ComponentDefaultPurpose
Virtual MailboxVirtualPostMailScans physical mail; sends notification email with PDF
Email (notifications)GmailReceives VPM scan emails; triggers PDF download
Email (bill intake)GmailReceives biller PDF attachments directly
OCRpdfplumber + pytesseractExtracts text from PDFs; auto-selects best method
LLMGemini 1.5 FlashClassifies documents; extracts sender, amount, due date
StorageGoogle DriveStores PDFs and JSON data (cloud-redundant)
SpreadsheetGoogle SheetsGenerated view layer; rebuilt from JSON on demand
FinanceYNABPulls bank transactions for bill reconciliation
Web DashboardFlask + HTMXThis UI — localhost:5000

Daily Pipeline (2am)

  1. Check email for VPM scan notifications → download PDFs → upload to /Inbox
  2. Check bill intake email for biller PDF attachments → upload to /Inbox
  3. Per PDF: OCR → LLM classify → rename → move to correct folder → update JSON
  4. Sync spreadsheet view from JSON
  5. Check storage consistency
  6. Pull bank transactions from finance provider
  7. Run bill matching (amount + date)
  8. Run entity discovery (fuzzy alias matching)
  9. Send daily summary email; immediate alert for ForwardToMe items

Requirements

  • Python 3.12+ on Windows 11 (macOS/Linux may work with minor adjustments)
  • A virtual mailbox service (VirtualPostMail, Earth Class Mail, Traveling Mailbox, PostScan)
  • A Gmail account (or other supported provider) to receive mailbox notifications
  • A Google account for Drive + Sheets storage (or configure an alternative)
  • An API key for a supported LLM (default: Gemini 1.5 Flash — free tier)

Step 1 — Clone and install

git clone https://github.com/PostMule/app.git PostMule
cd PostMule
python -m venv .venv
.venv\Scripts\activate
pip install -e .

Step 2 — Create config files

copy config.example.yaml config.yaml
copy credentials.example.yaml credentials.yaml

Both files are in .gitignore and will never be committed.

Step 3 — Set up Google OAuth

  1. Go to console.cloud.google.com and create a project
  2. Enable the Google Drive API and Google Sheets API
  3. Create an OAuth 2.0 Client ID (Desktop app type)
  4. Download the JSON file and save it as credentials.json in the PostMule directory
  5. On first run, PostMule will open a browser for OAuth consent — approve access

Step 4 — Encrypt credentials

postmule set-master-password
postmule encrypt-credentials

Your master password is stored in the Windows system keyring (DPAPI). After encrypting, you can delete credentials.yaml.

Step 5 — Test and schedule

# Test with a dry run (no writes, no emails)
postmule --dry-run

# Schedule the daily 2am run
postmule install-task

# Run immediately
postmule run

First-Run Checklist

  • postmule --dry-run completes without errors
  • Dashboard loads at localhost:5000
  • Storage folders created in Google Drive under "PostMule"
  • Test email received at your alert_email
  • Task Scheduler task visible in Windows Task Scheduler
